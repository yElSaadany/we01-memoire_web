Conclusion

Récapitulatif

Dans ce mémoire, nous avons tout d'abord présenté les points importants des sites web d'aujourd'hui, telles que des pages web de plus en plus lourdes et qui consomment de plus en plus d'énergie. Cela nous a permis d'introduire une solution à ce problème : le mouvement low-tech, un mouvement qui s'oppose au high-tech et se limite au strict minimum pour répondre aux besoins. Ce mouvement, appliqué aux sites web, était le sujet de ce mémoire.

Pour expliquer l'augmentation de la taille des pages web, nous avons présenté une brève histoire du développement web, avec la naissance des langages comme HTML en 1991, PHP en 1994, JavaScript en 1995, CSS en 1998. En effet, lorsque les langages comme JavaScript et PHP deviennent populaires, nous avons plus de fichiers de code dans le projet donc la taille d'un site web grandit. Ajoutons à cela l'augmentation des images et vidéos hébergées sur des sites web et on comprend mieux cette augmentation.

Nous nous sommes donc concentrés sur la taille des sites web, car plus un site web est lourd, plus le temps de chargement est lent, car on doit télécharger tout le code (et les ressources audiovisuelles) pour pouvoir afficher la page web. Or dans l'idée low-tech, plus un site web est léger mieux c'est (en parlant des choses superflues qui n'améliorent pas substantiellement le contenu). Pour diminuer la taille d'un site web, nous avons proposé des pistes aux développeurs. Il y a des idées qui tournent autour des habitudes à changer (comme utiliser du code externe) et des idées plus techniques (comme la compression d'image).

Dans la suite, nous avons décrit comment construire un site web low-tech et le rendre accessible publiquement. Pour cela nous avons étudié les différents types de site web existants, cela vous donne une première idée de quel type de site web que vous voulez créer. Par exemple, si vous voulez montrer vos produits, il vous faut un site vitrine, si vous voulez faire de la documentation, un site wiki est votre choix idéal, si vous voulez poster transmettre des informations dans l'entreprise, vous avez accès à un site intranet. Ces choses-là sont importantes, car la spécialisation permet d'améliorer la spécificité du code : l'exemple classique serait Wordpress, il n'est peut-être pas nécessaire d'utiliser Wordpress dans son intégralité lorsque l'on souhaite simplement publier des articles sur le Web (sans base de données, etc).

Après avoir présenté les différents types de sites web, nous avons identifié des services gourmands dans un site web, ce sont des services à éviter dans le cadre low-tech, ces services sont les images de hautes résolutions, le HTML généré côté client avec des frameworks comme React, les API, les publicités, etc.

Avant de créer notre propre site web, nous devons savoir comment le rendre publique, en effet, il faut l'héberger dans un serveur, pour cela nous avons proposé trois façons de faire : l'hébergement dans une société spécialisée dans les sites web, la location d'un serveur à distance (VPS) et l'auto-hébergement, dans le cadre low-tech, les deux dernières façons sont plus favorables, mais viennent au prix de devoir s'en occuper plus personnellement.

Pour les plus pressés, le chapitre sur les commandements permet d'avoir l'idée générale. Vous pouvez désormais passer à la phase de développement, en appliquant tous les points abordés. Pour vous inspirer, nous avons aussi donné des exemples de sites low-tech comme le site de The Green Emporium ou Low Tech Magazine pour vous donner des idées de ce que vous pourriez produire comme site web, ainsi un outil Pingdom qui vous permet de tester votre site web et les points à améliorer.

Pour aller plus loin...

Tout au long de notre mémoire nous avons appliqué le low-tech à des sites web. Mais, la philosophie low-tech ne se limite pas qu'aux sites web, on pourrait aussi bien l'appliquer dans notre vie quotidienne et on laisse au lecteur l'exercice de trouver ce qui est "low-techisable" dans son environnement.
