Conclusion

Récapitulatif

Dans ce mémoire, nous avons tout d'abord présenté les enjeux des sites web d'aujourd'hui, tel que des pages web de plus en plus lourdes et consomment de plus en plus d'énergie. Nous avons donc introduit une solution à ce problème, le mouvement low-tech, le thème de notre mémoire, un mouvement qui s'oppose au high-tech, on se limite au strict minimum pour répondre aux besoins.

Pour expliquer l'augmentation de la taille des pages web, nous avons présenté une brève histoire des sites web, avec la naissance des langages comme HTML en 1991, PHP en 1994, JavaScript en 1995, CSS en 1998. En effet, lorsque les langages comme JavaScript et PHP deviennent populaires, nous avons plus de fichiers de code dans le projet donc la taille d'un site web agrandit.

Ensuite, nous nous sommes concentrés sur la taille des sites web, car plus un site web est lourd, plus le temps de chargement est lent, car on doit télécharger tout le code pour pouvoir afficher la page web. Or dans l'idée low-tech, plus un site web est léger plus c'est mieux. Pour diminuer la taille d'un site web, nous avons proposé des pistes aux développeurs.

Dans la suite, nous avons décrit comment construire un site web low-tech et le rendre accessible publiquement. Pour cela nous avons étudié les différents types de site web existants, cela vous donne une première idée de quel type de site web que vous voulez créer. Par exemple si vous voulez montrer vos produits, il vous faut un site vitrine, si vous voulez faire de la documentation, un site wiki est votre choix idéal, si vous voulez poster transmettre des informations dans l'entreprise, vous avez accès à un site intranet.

Après avoir présenté les différents types de sites web, nous avons identifié des services gourmands dans un site web, ce sont des services à éviter dans le cadre low-tech, ces services sont les images de hautes résolutions, le design responsive, le HTML généré côté client avec des framworks comme React, les API, les publicités, etc.

Avant de créer notre propre site web, nous devons d'avoir savoir comment pour le rendre publique, en effet, il faut l'héberger dans un serveur, pour cela nous avons proposé 3 façons de faire, l'hébergement dans une société spécialisée dans les sites web, la location d'un serveur à distance (VPS) et l'auto-hébergement, dans le cadre low-tech, les deux dernières façons sont plus favorables.

Après avoir vu quelques commandements, vous pouvez désormais passer à la phase de développement, en appliquant tous les points abordés, nous avons aussi donné des exemples de sites low-tech comme le site de The Green Emporium ou Low Tech Magazine pour vous donner des idées de ce que vous pourriez produire comme site web, ainsi un outil Pingdom qui vous permet de tester votre site web et les points à améliorer.

Pour aller plus loin...

Tout au long de notre mémoire nous avons appliqué le low-tech à des sites web. Mais, la philosophie low-tech ne se limite pas qu'aux sites web, on pourrait aussi bien l'appliquer dans notre vie quotidienne. Voici quelques exemples :

  • prendre le transport en commun au lieu de sa voiture pour aller au travail pour réduire l'émission de CO2.
  • éteindre les appareils électriques en sortant de chez soi pour économiser l'énergie électrique.
  • aller au supermarché régulièrement pour acheter des produits en petites portions pour éviter le gaspillage.
  • préférer des produits recyclables.

Ce sont des gestes plus ou moins faciles à adopter, mais elles améliorent notre environnement.
