Exemples de site low-tech

Nous avons vu dans le chapitre précédent le développement et l'hébergement d' un site low-tech, et nous avons vu dans le chapitre 4 les différents types de site web. Donc dans ce chapitre nous allons donner quelques exemples concrètes de sites types de différents types qui sont low-tech, et les comparer à des sites web normals.

Outil : Pingdom[1]

Avant de présenter les exemples des sites low-tech, nous allons d'abord présenter Pingdom, un site web qui offre une surveillance rentable et fiable de la disponibilité et des performances de site web. Nous allons utiliser ce site web pour comparer les sites normals et les sites low-tech.

Tout d'abord ce site évalue le site que nous avons donné, il nous renvoie une note globale sur la performance, la taille de la page, le temps de chargement. Ensuite il nous offre des notes (allant de 0 à 100 et de F à A) sur des critères permettant d'améliorer la performance. Les critères sont :

  • Ajouter des en-têtes Expires
  • Compresser les composants avec gzip
  • Faire moins de requêtes HTTP
  • Utiliser des domaines sans cookies
  • Réduire les recherches DNS
  • Éviter les redirections d'URL
  • Éviter les src ou href vides

Il nous indique la taille de chaque type de contenu comme les vidéos, les images, les scripts, le code HTML, le code CSS, le font, les liens de redirections, etc.

Les sites vitrines

Comme nous avons vu au chapitre 4, un site vitrine est alors un site web qui expose ce que propose différents partis comme des PME, des artisans ou des entrepises de services. Prenons le même exemple pour un site normal, le site vitrine de BenjaminCode[2], nous allons le comparer avec un site vitrine low-tech, le site de The Green Emporium[3].

le site vitrine de BenjaminCode[2]

Niveu de performance : 74 C
Taille de la page : 13.5 MB
Temps de chargement : 786 ms


le site de The Green Emporium[3]

Niveu de performance : 84 C
Taille de la page : 799.8 MB
Temps de chargement : 286 ms



Nous observons qu'en effet un site vitrine low-tech est plus performant, plus petit en taille, et le temps de chargement est plus court. Cependant il reste toujours des points à améliorer pour avoir une meilleure performance.
Ensuite, pour les différents critères de performances, nous voyons que le site low-tech a des meilleures notes que le site normal.
Enfin, en observant les différents contenus, nous pouvons constater que les contenus les plus lourds sont bien les vidéos (XHR), les images et les scripts.
